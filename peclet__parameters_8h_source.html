11 /*
12 
13  @brief Encapsulates parameter handling and paramter input file handling.
14 
15  @detail
16 
17  Originally the ParameterReader from deal.II's step-26 was used;
18  but that was too prohibitive. The approach in this file isolates the details
19  of the input file handling from the rest of the program,
20  allowing for rich data structures and simplifying the user code.
21 
22  The goal is to allow one to run the program with different parameters,
23  without having to recomplile the program.
24 
25  It is valuable to structure the parameter data as done here, to greatly
26  simplify the writing and debugging of the code.
27 
28  This also makes it simple to insantiate a PDE Model in a user program
29  and then change it's parameters directly without having to use any intermediate text files.
30 
31  @todo Allow for multiple parsed boundary functions.
32  This might not be possible.
33  This will obsolete the "constant" function option.
34 
35  @author A. Zimmerman <zimmerman@aices.rwth-aachen.de>
36 
37 */
